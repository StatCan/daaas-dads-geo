    Using your Cloud VM

    About your GEO VM

    A GEO Virtual Machine will be created for you.

    Info

    Depending on your project, you may have the ability to create a standard virtual machine via the CAE, or create a workspace in the AAW. These self-serve VMs are NOT optimized for geospatial rendering.

    Learn more about CAE VMs

    Learn more about AAW Workspaces

    The standard GEO VMs include the following:

    Hardware: Software:
    - 4vcpus, 14GiB Memory
    - AMD Radeon Instinct MI25 GPU
    - learn more- Microsoft    		 - ArcGIS Pro (BYO-license)
    - QGIS
    - Anaconda
    - LibreOffice
    Plus resources identified during the Project Onboarding.

    How do I request changes to my virtual machine?

    If your VM does not meet your requirements, please contact the support team through the Jira GEO Support Board (NET B).

    Common Change Requests: - I need more capacity
    - I need additional software/packages
    - I need to update software
    - I need the auto-shut down adjusted
    - I need an auto-start enabled

    See the FAQ section for more information regarding Project Amendments.

    Finding a VM

    1. From the Geospatial Platform Azure Dashboard locate the Virtual Machine Tile
    2. Scroll down until you see your VM.
    3. Click on your VM Name to access its Overview page.

    Temporary_image--your__view_may_look_different

    Start Your Virtual Machine

    1. From the Overview page for your VM, click on the Start button.

    2. It takes a few minutes for your VM to start up. Monitor its start-up progress by selecting the Notifications icon at the top right of the window.

    Connect To Your Virtual Machine

    1. Start your VM, and wait for the notification (this can take a few minutes)

      • From the Overview page for your VM, click on the Browser connect button

      • If you do not see a Browser connect button, click on the Connect button and then choose Bastion from the dropdown menu.

    3. Ensure the Open in new window checkbox is selected, enter the Username and Password that were provided to you, and click on the Connect button. Your VM should open in a new browser tab.

    Info

    After attempting to login for the first time, an error may appear that a popup blocker is preventing a new window to open. To disable it, an icon will pop up on the browser's search bar, select the button and click always allow.

    Stop Your Virtual Machine

    Virtual machines only incur costs while they are running. You should shut down your virtual machine when not in use to prevent unnecessary charges. Your VM has an automatic shut-down time of 7pm PST.

    1. From the Overview page for your VM, click on the Stop button.

    Common Errors & Fixes

    Error Solution
    Your VM is still starting; wait a minute and try again
    You likely entered the VM username or password incorrectly, try again.
    You are experiencing some network latency; this usually resolves itself. If not, try restarting your VM

    Tips

    Using keyboard shortcuts while connected to a VM may not result in the same action as expected. (i.e. ctrl + z will close your main browser window, not the VM browser window)

    VMs have an auto-shutdown time of 7pm cst. (this can be adjusted upon request: FAQ)

    Moving the taskbar to the top or side of the VM might help differentiate between your OZ-PC and VM workspace.

    You can not alter the VMs display settings, however there is a way to almost use multiple monitors. Use the Restore Down button in your browser window and then drag to resize the window across multiple displays to get a multi-screen effect. You may need to adjust some configurations in your OZ-Laptop display settings.

    By default, the Ubuntu virtual machine opens in Terminal mode. You can access the GUI of your Ubuntu machine from a Windows machine using X2Go.
